- **Настраиваемая звуковая тревога** — в админке выбирается, какие типы сообщений (ракета/БПЛА/отбой/...) и для каких районов присылают громкий push (со звуком и вибрацией на телефоне); можно полностью отключить звуковую тревогу.
- **Админка** (`/admin.html`) — статистика посещений (как мини-Метрика) и ручное управление лентой: отбой по району или сразу по всей области, произвольные сообщения любого типа. См. раздел «Админка» ниже.

## Админка (`/admin.html`)
Откройте `https://ваш-сайт/admin.html` — потребуется пароль.

**Пароль администратора**: задайте `ADMIN_PASSWORD` в переменных окружения хостинга — это самый надёжный способ. Если не задать, сервер сгенерирует временный пароль и **сохранит его на диск** (`data/admin-password.json`), поэтому он переживёт обычные перезапуски/«засыпания» процесса (посмотреть значение — вкладка **Logs** на хостинге сразу после первого старта). Он изменится заново только если хостинг полностью стирает файловую систему при новом деплое (это отдельная особенность некоторых бесплатных тарифов, не баг самого приложения) — поэтому для продакшена всё равно лучше один раз зафиксировать `ADMIN_PASSWORD` явно.

**Что показывает дашборд:**
- Всего визитов, уникальных посетителей, визитов сегодня, активных push-подписчиков
- График визитов по дням (30 дней) и по часам за сегодня
- Разбивка по устройствам (моб./десктоп/планшет) и браузерам
- Источники переходов (реферреры)
- Подписчики push в разбивке по районам
- Сообщения в ленте по типам
- Статус каждого подключённого Telegram-канала (жив/ошибка, сколько сообщений забрал за последний опрос)
- Таблица последних посещений сайта

**Ручное управление:**
- **Быстрый отбой** — кнопки на каждый район плюс «Отбой по всей области» (уходит всем подписчикам независимо от выбранного ими района)
- **Написать в ленту вручную** — выбор типа сообщения (ракетная опасность / БПЛА / отбой / укрытие / повторно / произвольное объявление администрации), района (или «по всей области») и текста. Сообщение сразу появляется в ленте у всех пользователей и рассылается как push-уведомление, помечено бейджем «вручную», чтобы отличаться от сообщений парсера.
- **Источники (каналы)** — добавление нового Telegram-канала по ссылке (`https://t.me/имя`, `t.me/s/имя`, `@имя` или просто `имя`) и удаление существующих (должен остаться минимум один).
- **Звуковая тревога** — чек-бокс «включена/выключена» целиком, плюс выбор конкретных типов сообщений и районов, для которых push будет громким (со звуком, вибрацией и требованием реакции на экране). Остальные типы/районы приходят как обычное тихое уведомление.

Админка не индексируется поисковиками (`robots.txt` и мета-тег `noindex`), но ссылки на неё нет на самом сайте — открывать напрямую по адресу `/admin.html`.

## ⚠️ Важно про хостинг
Обычный статический хостинг (просто «закинуть файлы») подойдёт **только для `public/`**, но парсер и push-рассылка — это `server.js`, который должен работать **постоянно** (как процесс Node.js), а не просто отдавать файлы. Статический хостинг (например, обычный shared-хостинг или GitHub Pages) для этого не годится.

### Деплой на Render — в один клик через Blueprint
В проекте уже есть `render.yaml`. Самый быстрый способ:
1. Залейте проект в свой репозиторий на GitHub.
2. На Render: **New** → **Blueprint** → выберите репозиторий.
3. Render сам создаст сервис по `render.yaml` и **сгенерирует надёжный `ADMIN_PASSWORD`** (посмотреть — Dashboard → сервис → **Environment**). Он больше не будет меняться сам по себе.
4. Сгенерируйте VAPID-ключи локально (см. раздел ниже) и впишите их в **Environment** на Render — без них push-уведомления не заработают, но сайт и лента будут работать и без них.
5. Деплой запустится автоматически. Через минуту сайт будет доступен по адресу вида `https://belgorod-alert.onrender.com`.

Другие варианты хостинга (без Blueprint):
1. **Всё вместе на одном Node-хостинге** (проще всего): Render.com, Railway.app, Fly.io — все имеют бесплатный/дешёвый тариф с поддержкой постоянно работающих Node-процессов. Сервер сам отдаёт и `public/`, и API — грузите весь проект целиком.
2. **VPS** (timeweb, reg.ru, Selectel и т.п.) — запустить `node server.js` через pm2/systemd, чтобы процесс не останавливался.
3. Если у вас уже есть только статический хостинг для фронтенда — задеплойте `server.js` отдельно (вариант 1 или 2), а в `public/index.html` впишите его адрес в константу `API_BASE`, например:
   ```js
   const API_BASE = 'https://ваш-backend.onrender.com';
   ```

## Настройка push-уведомлений (VAPID)
Сгенерируйте постоянную пару ключей (иначе при каждом перезапуске сервера все подписки слетят):
```bash
npx web-push generate-vapid-keys
```
Задайте их как переменные окружения на хостинге:
```
VAPID_PUBLIC_KEY=...
VAPID_PRIVATE_KEY=...
ADMIN_PASSWORD=ваш-пароль-для-админки
```
Если их не задать, сервер сгенерирует временные ключи сам и выведет их в консоль при старте — это нормально для теста, но не для продакшена.

## Telegram-бот — резервный канал (рекомендуется)
Веб-push идёт через серверы Google/Apple — при ограничениях мобильного интернета
(например, во время атак, когда работает только «белый список» Минцифры) push может
не доходить. Telegram обычно остаётся на связи стабильнее, поэтому в проект добавлен
бот как дублирующий канал доставки — работает независимо от push, ничего не нужно
согласовывать заранее.

1. В Telegram напишите `@BotFather` → `/newbot`, следуйте инструкциям, получите токен
   вида `123456:ABC-...` и имя бота вида `trevoga_belgorod_bot`.
2. Задайте переменные окружения на хостинге:
```
TELEGRAM_BOT_TOKEN=123456:ABC-...
TELEGRAM_BOT_USERNAME=trevoga_belgorod_bot
```
3. После деплоя на сайте в настройках уведомлений (значок 🔔) появится кнопка
   «✈️ Подписаться в Telegram». Пользователь жмёт `/start` в боте, выбирает район —
   и получает сообщения в Telegram при каждой ракетной опасности/БПЛА/отбое, теми же
   вызовами, что и парсер/ручная отправка/быстрая тревога в админке — отдельно
   ничего не нужно дублировать.
4. Без этих двух переменных бот просто не запускается — сайт и push работают как обычно.

### Админ-режим в самом боте
Тому же боту можно дать права администратора прямо в переписке — без захода на сайт:
- `/admin ваш_пароль_админки` — вход (тот же пароль, что и в веб-админке). Сообщение с
  паролем бот сразу удаляет из чата.
- `/stats` — та же статистика, что на вкладке «Статистика» в админке: визиты,
  подписчики (push + Telegram), лента, статус парсера, последние ошибки из логов.
- `/alert` — кнопками отправить ракетную опасность / БПЛА / отбой по любому району —
  уходит в ленту и всем подписчикам той же логикой, что и кнопки в веб-админке.
- `/adminlogout` — выйти из режима администратора в этом чате.

## Push на iPhone — важные ограничения Apple
- Работает **только в Safari**, начиная с **iOS 16.4**.
- Сайт должен быть **добавлен на экран «Домой»** («Поделиться» → «На экран «Домой»») и **открыт хотя бы один раз** как отдельное приложение — из обычной вкладки Safari подписка на push недоступна.
- После этого разрешение на уведомления запрашивается так же, как в браузере — через кнопку «Включить уведомления» в приложении.

## Push на Android
Работает в Chrome/Edge/Firefox как в обычной вкладке, так и после установки на домашний экран — без дополнительных ограничений.

## Как работает парсер
- Каждые 10 секунд сервер запрашивает `https://t.me/s/mchs31` (открытая веб-версия канала, авторизация не нужна).
- Сообщения классифицируются по ключевым словам: ракетная опасность / БПЛА / отбой / укрытие / повторно.
- Регион определяется по упоминанию города/района в тексте; если не найден — по умолчанию «Белгород».
- Сообщения, похожие на рекламу (по ключевым словам и признакам), отбрасываются.
- Новые сообщения сразу уходят подписчикам через Web Push с учётом выбранных ими районов.

## Ограничения парсера, о которых стоит знать
- Классификация по ключевым словам — эвристика. Если МЧС31 изменит формулировки, часть сообщений может попасть в категорию «Сообщение канала» вместо конкретного типа — стоит периодически сверяться с реальной лентой и донастраивать `REGION_KEYWORDS` / классификатор в `server.js`.
- `t.me/s/<channel>` — неофициальный, но общедоступный способ чтения каналов; если Telegram изменит вёрстку этой страницы, потребуется поправить селекторы в `fetchChannelMessages()`.
